JavaScript Variables

Variables are names given to a memory location to store data in a program to use it in that program where needed. Simply a variable is a container for a value (data).

NOTE : Variables are not values, they are container for values.

Variable Naming Rule

Variable names -

  • can contain letters (A-Z, a-z), numbers (0-9), dollar sign ($) and underscore (_).
  • Must start with a letter, underscore or dollar sign.
  • Must not start with a number.
  • Cannot use reserved words e.g. let while etc.
  • Must be a single word, cannot use space (or any whitespace)
  • JavaScript allows naming variables using most of unicode characters, but it is not recommended

NOTE : Best practice is to use only characters at the beginning of a variable name, using lowerCamelCase

Declaring a Variable

To use a variable in a program we have to declare it first, to let the JS Engine know we have to allocate a memory space to store a data in it.

There are 3 types of variable declaration in JavaScript. Using var, let and const.

var

Variables declared using var keyword are function-scoped (can only be accessible in a function in which it is declared) or global-scoped (can be accessible everywhere in the program).

If we declare variables using nothing, for example a = 100 JavaScript will not throw any error. These variables will be treated as global variable. Unlike var declared variables, Memory allocation of these variables happens while executing. So if we want to access this variables before initializing, JavaScript will throw an error (ReferenceError) . Because JavaScript cannot find anything as that name of variable in memory (any scope). It is not recommended to use this type of declaration.
