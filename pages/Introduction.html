History of Javascript

JavaScript was created by Brendan Eich in 1995 just in 10 days. Became the ECMA-262 standard in 1997. It was originally named Mocha and later known as JavaScript. It was developed for Netscape. Now Mozilla foundation continued to develop JavaScript for the Firefox browser.

Javascript Engine

It is a software or program that executes JavaScript code. JavaScript engine is a part of modern browsers, when javascript was invented, it was a interpreted language. But now modern browsers use just in time (JIT) compilation to improve performance. JavaScript engine is used not only in browsers but also it is a core component of Node.js. Example of some famous JavaScript engines are -

  • V8 engine by Google used in Google Chrome
  • Spider monkey by Mozilla used in Firefox Browser
  • JavaScriptCore by apple used in Safari Browser
  • Chakra by Microsoft used in Internet Explorer
  • Chakra was used by Microsoft Edge browser, but now it uses V8 engine

What happens in JavaScript engine?

JavaScript engines convert high level code into machine readable code to execute the code. Different JavaScript uses different techniques to execute JavaScript code. Some engines use interpreter and compiler both (e.g. V8 engine). As an example, Google chrome's JavaScript V8 engine works like this -

First JavaScript source code goes to Parser manly it checks for syntax. Breaks the codes into token.

Now these tokens are converted to to AST(Abstract Syntax Tree) it allows interpreter to read the code and understand.

Now AST goes to Interpreter to get converted to Byte code. In V8 engine it is known as Ignition

If there is repeating code, it will decrease the performance of interpreter. Profiler will check repeating codes. To increase performance Profiler moves repeating code to compiler.

It generates byte code. And Byte codes execute and do specific task. In v8 engine this compiler is called as TurboFan.
